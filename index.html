My Retirement Plan 

    Instructions                    

    This spread sheet is provided to assist in understanding and planning future or current retirement.  Multiple tabs providing several features are included to enable one to collect categories of information.  These categories include Assets and Liabilities, Investments, Budget, Retirement, and Useful Facts.  Information about each of these tabs follows.                  

    (1) Instructions  -   the tab being read.  Describes how to use this spread sheet and enter data to produce the retirement finance  model.              
    (2) Assets & Liabilities  -    use this tab to define primary assets and liabilities.  Entries from this tab populate portions of the Investments and Budget tabs.  This tab also provides high-level overview net and gross worth.             
    (3) Investments  -  use this tab to estimate future investment contributions and rate of returns.  Up to six investments that are entered on the Assets & Liabilities tab can be defined.  This tab is also used to define what is defined as 'House Buy Down'.  If payments are being made for a mortgage the estimated monthly buy down of the mortgage can be entered.  This monthly amount is totaled for the year and that total is added to the total value of all investments.               
    (4) Budget  -   this tab is used to define the current monthly budget.  The defined 'Inflation Rate' from the Retirement tab is used to calculate the estimated budget for the target retirement start date.  This start date is also obtained from the Retirement tab.                 
    (5) Retirement  -  this tab is where data is entered and the retirement modelling happens. Several input fields are available to define the Income, Budget, and Withdrawals that will be used for each year of retirement.  These fields also allow defining Social Security benefits and housing related data.  Modeling and What-If can be performed on this tab.  Up to five different scenarios can be created, saved, and loaded.              
    (6) Useful Facts  -   this tab provides useful facts regarding past inflation, consumer price index, Social Security cost of living adjustments, federal tax rates , etc.  This data can help determine what is a valid or logical value for some of the entry fields on the Retirement tab.                
    Detailed information and instructions for each of the above mentioned tabs is provided in the following sections.

These categories, Cash, Real Estate, Vehicle, Other Assets, Investments, and Other Debts can be defined with descriptions, account information, balances, value, and monthly payments.     Budget                  

    "This tab provides the ability to capture the current budget in eleven categories.  Multiple fields are link into this tab from the Assets & Liabilities and Retirement tab.  Fields from Assets & Liabilities are included categories Housing, Transportation, Debt Payments.  Data from the Retirement tab includes the '(11) Yrly Inflation Rate %' which is used to define the field Inflation Rate.  The field 'Years till retirement' is calculated based on the field '(1) Start Date' from the Retirement tab.

For each line item of the budget four fields can be defined a.) Description, b.) Annual Freq., c.) Inflation Applies, d.) Expense Amount. Example Descriptions have been provided, yet can be changed as needed. The field 'Annual Freq.' defines the number of times the expense occurs for the year. Valid values are 0 to 366. Inflation Applies will control if this expense should have inflation calculated for each year till retirement. Expense amount is the dollar amount for the defined expense and frequency.

Two additional fields can optionally defined for each line item. The '(+/-) Monthly Adjust at Retirement' field is used to increase or decrease the line item when the target year for retirement is reached. If this field contains a positive or negative value the cell will be color coded. Green for positive amounts and Red for negative amounts. "

Entering data

The following image shows the primary entry fields. Following the image is a description of each field along with comments on usage as appropriate. Notice that fields have an identifier at the beginning of the field title e.g. Id1, Id2, 1, 2, etc. This identifier is used in the following descriptions." 

    Id/#    "Req

(or) Opt" Comments
Id1 Req Comprised of two entry fields, Name and Date of Birth. The Date of Birth is required while the Name field is optional. The Date of Birth is used to calculate Age and Social Security Benefits.
Id2 Opt Use these fields if a second individual is to be included in the model. Id2 as with Id1 is comprised of two entry fields, Name and Date of Birth. The Date of Birth is required while the Name field is optional. The Date of Birth is used to calculate Age and Social Security Benefits.
1 Req The estimated date to start retirement using a mm/dd/yyyy formatted data. This date is used on multiple tabs. On the Budget tab this date will be used to calculate the future budget balance at start of retirement. On the Investments tab this date is used to determine which Balance is to be used as to populate field (9) Starting Retirement Savings.
2 Opt Enter a yearly tax rate percentage between 0.00% to 99.99%.  This should include all taxes as a single value  e.g. Federal, State, County, City, etc.  This percentage is used to calculate the amount of taxes to withhold from the Yrly Total Income. If no taxes are to be calculated the field can be left blank.
3 Opt Enter the estimated yearly rate of return on all investments, savings, etc. as a single percent. Valid range is 0.00% to 99.99%.
4-1 Opt For this Individual enter the estimated amount of the Full Retirement Amount (FRA). The FRA value is obtained from the Social Security Administration. Valid values are a whole number between $0 and $9999.
5-1 Opt For this Individual enter the date to start receiving Social Security benefits. Enter using format mm/dd/yyyy. Valid date range is 01/01/2000 to 12/31/3000. This date and the FRA are used to calculate (6-1) SS Calculated Start Amt.
6-1 N/A The calculated benefits to be received from Social Security based on the information entered in fields 4-1 and 5-1.
n-2 If a second individual (Id2) is entered the fields 4-2, 5-2, and 6-2 fields are used to define and calculate the Social Security benefits for the second individual. Use the same instructions as provided in 4-1 and 5-1 above.
7 Opt Enter the estimated Social Security COLA (Cost of Living Adjustment) as a percentage.  Valid range is 0.00% to 99.99%. Refer to the Useful Facts tab for past annual COLA amounts.
8 Opt Enter the year that one individual will stop receiving Social Security benefits.  Enter the year as yyyy using a whole number between 2000 and 3000, or blank.
9 N/A Amount is linked from the Investments tab based on the year of the date entered in field (1) Start Date.
10 N/A Amount is linked from the Budget tab based on the year of the date entered in field (1) Start Date.
11 Opt Enter the estimated yearly inflation rate as a percent.  Valid range is 0.00% to 99.99%. Refer to tab Useful Facts for past annual inflation rates.
12 Opt Enter the Yearly Fixed Additional Annual Income amount.  Valid value is a whole number between $0 and $5,000,000. Example usage would be the total annual amount of an annuity.
13 Opt If field (12) has been entered use this field to enter an annual percent of increase for Fixed Additional Annual Income if applicable.  Valid value is a percentage between 0.00% and 99.99% or blank.
14 N/A Amount is linked from the Investments tab based on the year of the date entered in field (1) Start Date.
15 Opt Enter the estimated yearly appreciation rate of house.  Enter as a percentage between 0.00% and 99.99%.
16 N/A Amount is linked from the Budget tab based on the year of the date entered in field (1) Start Date.
17 Opt If a house is owned an a decision is made to sell the house once retired enter the estimated year in this field.  Valid value is a whole number between 2000 and 3000 or blank.
18 Opt Enter the estimated monthly increase in housing / apartment expense after house is sold.  Valid value is a whole number between $0 and $25,000. 
19 Opt Enter the estimated yearly percent of increase for housing cost after house is sold.  Valid value is a percentage between 0.00% and 99.99%. This field allows housing cost to adjust at a different rate than the inflation rate entered in field (11).
25 Opt If house mortgage is paid off after starting retirement enter the year as yyyy when paid off.
26 N/A The amount of the monthly mortgage payment as entered on the Assets & Liabilities tab. If the mortgage is paid off after starting retirement this amount will be subtracted from the monthly budget base on the input from field (25).
Fields 20 - 24 are available to be entered for each year of retirement.
20 Opt Enter supplemental income for the year.  Valid value is a whole number between $0 to $5,000,000.  Examples: Income from and inheritance, sale of second vehicle, etc.
21 Req Press 'Calculate Withdrawals' button to auto calculate amount for this column or enter the estimated withdrawal needed for this year to cover the Yrly Total Budget. 
22 Opt Enter annual supplemental withdrawal amount.  Valid value is a whole number between $0 and $5,000,000.  Examples: Increased insurance till Medicare age is reached, additional travel /vacation, buy vehicle, etc.
23 Opt Enter percentage of budget change from the previous year.  Valid value is a percentage between negative -99.90% and positive +99.99%.   A negative percentage value will decrease the budget, a positive percentage number will increase the budget.  
24 Opt Enter comment to help explain one or more entries for this year.

        Calculate Withdrawal (button)               

        "Calculating the amount to withdrawal from saving to match the required budget amount is automated when using the button labeled 'Calculate Withdrawal'.

As shown on the left hand side below there are several years when the amount of money withdrawn is inadequate to meet the amount needed for the budget. The inadequate amounts are shown in field labeled 'Mlty Additional Budget Needed' with a yellow background and red dollar amount in parenthesis. The left hand side below also indicates there are two years when excess money is being withdrawn, fields with green background and dollars amounts greater than zero.

By pressing the 'Calculate Withdrawal' button the target withdrawal amount column labeled '(21) Yrly Minimum Withdrawal' is calculated. This process will take a few seconds and is not immediate. A pop-up message is displayed to indicate the time needed to complete the calculations. Ensure the OK button is pressed to continue once the message is displayed. The successful completion also results in a pop-up message indicating the amounts have been calculated. Press the 'OK' button to continue. The desired result is to have zero dollar amounts with green background color as shown on the right hand side image below.

Dollar amounts in the column labeled '(22) Yrly Supp Withdrawal' will not be modified then the calculate button is pressed. Amounts in this column provide a method to request additional withdrawal amounts above the minimum needed to meet the budget.

Note: There are occasions that the result from the calculation does not return all zero dollar amounts. If this occurs press the 'Calculate Withdrawal' button again to calculate from this base. The button may be pressed as many times as desired.

    "Up to five scenarios can be saved or loaded.  A scenario can be saved at any time while entering data on this tab.    When saving a scenario enter a brief description in the entry to be saved.  Once saved the background of the entry will change to green and the Status message with display 'Saved'.

When a scenario is loaded all existing data that is being shown in the entry fields will be replaced with the newly loaded scenarios data. The background for the scenario description will change to red with white text and the Status message will display 'Loaded'.

To indicate that a scenario has been successfully save the following pop-up message box is shown with the appropriate scenario number. Press the 'OK' button to continue. No pop-up message box is displayed when a scenario is loaded.
